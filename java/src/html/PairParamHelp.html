Alignment & Synteny Parameters

These parameters apply to the alignment, anchor processing, and synteny computation (AAS) for a pair of projects. The rules for saving and using parameters are a bit complex, as described in the last section; typically, the defaults will work fine.

The arguments for BLAT and MUMmer are NOT checked for correctness.

Rules for saving parameters

Setting parameters:
  • Selected Pair Parameters: If a project pair is selected from the matrix, then the parameters changed followed by "Save", the parameters will be saved to the database.
  • Global Pair Parameters: If no project pair is selected in the matrix, then the parameters are changed followed by "Save", these parameters will be applied to all project pairs that DO NOT have previously saved parameters, or with the "Globals" button. Global parameters are only restained for the session.
  • Otherwise, default parameters will be used.
Storage of parameters:
  • To see the project pair parameters, select the pair followed by "Parameter", which will show:
    1. If parameters are stored in the database for the pair, they will be shown.
    2. If no parameters are stored in the database for the pair, the Global parameters will be shown.
  • A "Remove project from database", "Reload project" or "Clear pair" clears the pair parameters from the database.
  • If the parameters are changed after AAS is run, and AAS is not re-run, then the parameters in the database will NOT correctly reflect what was used for the Pair's AAS computation.
