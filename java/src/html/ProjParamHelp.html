Sequence Project Parameters

This page provides all the setup parameters for a sequence project. The most important are the following:

sequence_files

Select the fasta-formatted sequence files, or directories of sequence files, to upload to the project. There must be at least one sequence file. Note that a file can contain multiple sequences. Default: data/seq/<project-name>/sequence (if no "sequence_files" are entered, SyMAP will look in the default directory).

anno_files

Select gff3-formatted annotation files corresponding to your sequences. SyMAP currently recognizes annotations of type gene,exon,centromere,gap, and CDS (treated the same as exon). The type is read from column 3 of the gff file. Annotation is optional but highly recommended. Default: data/seq/<project-name>/annotation (if no "anno_files" are entered, SyMAP will look in the default directory).

grp_prefix

  • The parameter effects the display, but is applied during the loading of the sequences. If provided, this allows SyMAP to remove the prefix from the chromosome or contig names and use the remaining part as a shorter name (e.g. "100" instead of "contig_100"). All sequences must have this prefix.
  • If the sequences have a mix of prefixes, leave this blank.
  • Long names really clutter the display; it is best to use a consistent prefix that SyMAP can remove, otherwise, use really short prefixes, e.g. 's' for scaffold, 'c' for chromosome.
  • NOTE: If you change this after load, you need to re-load and re-run all computations!

Other parameters

The following table is broken into section on what needs to be re-run following the parameter change.

Nothing needs to be re-run
category Category label for the project. This is only used in the Summary list of the Project Manager window. Uncategorized
display_name A reader-friendly name for the project. Can contain any characters and multiple words, but shorter names will work better in the displays. defaults to the name used to create the project (typically the directory name)
description Description of the project. This is only shown in the Project Manager Summary section.  
grp_type How to refer to the individual sequences, e.g. as "Chromosome" or "LG" or "Contig" Chromosome
min_display_size_bp For the dotplot display, in case some of your sequences are much shorter than others. Set this value to a minimum basepair size and all sequences will be scaled to appear at least this large shown in a dotplot. 0
Reload Project (remove existing alignment files), Alignment&Synteny
grp_prefix Explained above Chr
min_size Minimum length of sequence to load; smaller sequences will be ignored. Note that annotations for ignored sequences will also be ignored, but some error messages will print. 100000
Reload annotation, Alignment&Synteny (if annotation files exist, only Synteny will be run).
annot_keywords SyMAP parses keyword/value pairs from the attributes field (last column) of the annotation gff files, which are entered into the database as the "Annotation Description". The keyword/value pairs to be saved in the database can be limited by listing the desired keywords here (separated by commas).
annot_kw_mincount For the SyMAP Query, all keywords are search UNLESS they have less than this many occurrences, e.g. if only one gene has a given keyword, then it is not included on the SyMAP query search, however, it will still be in the gene's annotation description. 0
Remove alignment files, Alignment&Synteny
mask_all_but_genes Mask out all non-genic parts of the sequences before running MUMmer (gene annotation must be provided). This can save time but prevents non-annotated anchors from being found. no
Alignment&Synteny (if alignment files exist, only Synteny will be run).
order_against For draft contig sets, this allows you to order them using synteny to one of the other projects. The ordering occurs automatically after the synteny alignment is run, resulting in the following two:
  • A file showing the ordering is printed into the "order_against" project directory called "ordered.cvs".
  • A new "_ordered" project is created, in which the draft contigs are concatenated into chromosome sequences based on the synteny; these can be aligned to the "order_against" project in order to get a less fragmented alignment.
