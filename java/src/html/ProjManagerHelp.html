Summary Section

The projects selected from the Projects in the left panel are listed in this Summary Section. The possible functions vary with the state, as listed below:
FunctionDescription
If there is any project not loaded to the database, you will see:
Load All ProjectsLoad all projects that have not been loaded yet.
If a project is not loaded to the database, you will see:
Remove project from diskRemove your files from disk. The project will no longer be shown on the left.
Load projectLoads the sequence and optional annotations to the database.
ParametersThis brings up a panel of paramters. Make sure the parameters are set right before loading the project.
If a project is loaded into the database, you will see:
Remove project from databaseThe sequences and annotation will be removed, but the files stay on disk.

Available Synteny Section

This section shows a table with the status of alignments between the selected projects. (Only projects loaded to the database are shown.)

Each cell in the table represents a pair of projects (labeled on the rows and columns) and the cell contains a status code showing whether or not that pair has been aligned (codes are listed below). Note that the table actually shows each pair twice, but only the lower cells are activated.

Clicking on a cell selects that pair of projects (the cell then highlights in yellow). Alignment and synteny may then be computed or viewed for that particular pair, using the function buttons (described below). If no cell is selected, you can still use some of the function buttons, which apply to all the cells.

Status Codes:
Symbol Description
✔ Synteny for this project pair is ready to view.
A The alignment (MUMmer or BLAT) has been performed but the anchors have not been loaded and the synteny computation has not been run. This status occurs if a pair is completed but then annotations are re-loaded for one of the projects, or if the MUMmer files have been added by the user (see website "System Guide" for details).
? SyMAP puts an empty file with the name 'all.done' in the /align directory to indicate all alignments successfully completed. If this file is not present, the alignment may not have completed. In this case, any alignments will be completed and the synteny run.
<empty cell> The alignment has not been started.

CPUs parameter: Enter the maximum number of CPUs you wish to use for an alignment. SyMAP will use up to that number (but may use fewer, depending on the number of sequences being aligned). Alternatively, the number of CPUs may be entered as a command line argument (-p N) or entered in the symap.config file.

Function Buttons:
Label Description
Alignment & Synteny
Selected Pair Run (or complete) the alignment & synteny computation for the selected project pair. If the pair is already complete, the button label changes to "Re-do", and the function is to re-run the synteny computation only. If you wish to re-run the BLAT or MUMmer alignment also, first use the "Clear Pair" function.
All Pairs Run (or complete) the synteny computation for all pairs in the Available Syntenies table. However, this will not run 'self-syntenies'; those need to be done individually by selecting the pair.
Parameters Set alignment & synteny parameters (has its own help window).
Clear Pair Fully reset the selected pair, removing its synteny results and its MUMmer or BLAT alignment files.

Display for All Projects
Chromosome Explorer Show the Explorer, allowing selected chromosome sets to be viewed in dot plot, side-by-side, circular, or 3D displays (the latter only for supported platforms.)
Queries
SyMAP Queries Show the SyMAP Query interface, for locating shared genes, orphans, putative gene families, and other valuable synteny-derived information.
