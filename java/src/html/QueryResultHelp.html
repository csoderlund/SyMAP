SyMAP Query Results

This page shows the table of hits (anchors) resulting from the query, where each hit connects two species. If the query was for orphan genes, then each row represents one gene and shows data only for one genome.

Button functions

For selected: As detailed for the following two:

Align sequences
Select one or more rows: All sequences (two per row) are written out and a multiple alignment is created using MUSCLE (Edgar 2004 NAR:32) and shown in the main panel.

NOTE: This is especially useful to view the similarity of putative gene families (PgeneF), as follows: check "Compute PgeneF" and "Run Query", show the PgeneF and PgFsize columns, sort on PgeneF, then select the rows for PgeneF to align.

Show Synteny
Select one row: Brings up a SyMAP 2-track view for the selected entry. The hits are initially padded to each side by the margin amount indicated (default 50kb), but you can easily zoom further out in the 2-track views.

For table (or selected): One or more rows can be selected for the following two exports, or if no rows are selected, the table is exported.

Export CSV
Exports the rows using the selected set of columns to a CSV format suitable for import into Excel.

Export Fasta
Export the sequences from the selected hits in Fasta format. Both sides of each hit are written, using the start/end coordinates shown in the table.
NOTE: this is a very slow function and takes minutes if many rows are selected.

Results Table

The table shows the hits (or orphan genes) resulting from the query. Each row is a single hit or orphan gene. Note that the table contains columns for all of the selected species, but each hit only connects two species, and the other species columns are empty (and an orphan gene only has one non-empty species).

You can sort the columns by clicking on them, and rearrange them by dragging the header boxes. You can add/remove columns using the "Select Columns" button at the bottom. Following are the meanings of the columns:

General
Row   Row number within the table
PgeneF   If Compute PgeneF: PgeneF number. All hits in a given PgeneF have the same number. Note that the number is generated during the search and won't necessarily be the same in a different search.
PgFSize   If Compute PgeneF: Size of the PgeneF which contains this hit. Sort on this column to view the other rows with same PgeneF.
HitIdx   Database index of the hit. This is shown on the "Chromosome Explorer" when the mouse is over the hit line.
Block   Synteny block containing this hit (if any). The format is C1.C2.N, where C1 and C2 are the chromosome numbers and N is the block number for the two chromosomes.
Block Score   The number of hits which comprise the synteny block.
RunSize   If the hit is contained in an uninterrupted collinear set of shared genes, RunSize shows the length of the set. This column is used for the "Collinear gene" query.
Location: one row for each species
Chr   Chromosome (or draft sequence number) where the hit is aligned
Start/End   Start/end of the hit region.
NOTE1: Hits are clustered during anchor loading, so hit regions are not necessarily single MUMmer alignments).
NOTE2: If the table contains single project genes (e.g. orphan or all), then the coordinates are the gene coordinates.
Gene#   The order number along the chromosome of the gene. This is shown on the Chromosome Explorer "Annotation Description" and when the mouse hoovers over the gene.
Annotation: one row for each species
The keywords for the annotations of each species are listed. They can be different for each species, e.g. one may have "ID" while another uses "Name" for the gene names. "All_anno" shows all of the attributes joined together.

Statistics

This section shows overall statistics for the query results. Most of the statistics are self-explanatory except the following:

Annotated and Genes: The first is the number of hits that overlap one or more genes, where a gene can have multiple hits. The second is the number of Genes with at least one hit.

Regions: This statistic is only shown if Compute PgeneF was checked, and it is the number of distinct regions covered on that species.
