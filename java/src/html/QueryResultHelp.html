SyMAP Query Results

For more information, see User Guide.

The results table list the hits (anchors) resulting from the query, where each hit connects two species. If the query was for single genes, then each row represents one gene and shows data only for one genome.

Button functions

Selected: One row can be selected.

Show
A popup will show the information for all columns of the selected row.

Align
Select one or more rows: All sequences (two per row) are written out and a multiple alignment is created using MUSCLE (Edgar 2004 NAR:32) and shown in the main panel. The alignment is left in the muscle sub-directory and is over-written with each new alignment.

View 2D
Select one row: Brings up a SyMAP 2-track view for the selected entry.
Region (kb) The hits are initially padded to each side by the margin amount indicated (default 50kb), but you can easily zoom further out in the 2-track views. The default filter is "Show all hits".
Synteny block The entire block for this hit will be shown. The default filter is "Show Block Hits".
Collinear set If the hit is in a collinear set, the set will be shown. The default filter is "Highlight Collinear Set; Show Block and Set Hits".
High: If this is checked, the selected hit will be highlighted in the 2D display using the hover color.

Table (or selected): One or more rows can be selected for the following exports, or if no rows are selected, the table is exported.

Export CSV
Exports the rows using the selected set of columns to a CSV format suitable for import into Excel.

Export Fasta
Export the sequences from the selected hits in Fasta format. Both sides of each hit are written, using the start/end coordinates shown in the table.
NOTE: this is a very slow function and takes minutes if many rows are selected.

Results Table

The table shows the hits (or single genes) resulting from the query.
  • The table contains columns for all of the selected species, but each hit only connects two species, and the other species columns are empty.
  • If the hit aligns to a gene on either/both ends, columns of information about the gene(s) is included; however, if the hit aligns to overlapping genes, only the gene with the best overlap is provided.
  • If the results are for single genes, then only one species information is provided and the other species columns are blank.

Columns

You can sort the columns by clicking on them, and rearrange them by dragging the header boxes. You can add/remove columns using the "Select Columns" button at the bottom. Following are the meanings of the columns:

The "Single genes" tables only have the Gene Location and Annotation columns.
General
Row  Row number within the table
Block  Synteny block containing this hit (if any). The format is C1.C2.N, where C1 and C2 are the chromosome numbers and N is the block number for the two chromosomes.
Block Score  The number of hits which comprise the synteny block.
Collinear  Collinear set containing this hit (if any). The format is C1.C2.Size.Set (e.g. 1.2.5.100; there are 5 adjacent gene hits in set 100 on Chr1 to Chr2).
PgeneF  If Compute PgeneF: PgeneF number. All hits in a given PgeneF have the same number. Note that the number is generated during the search and won't necessarily be the same in a different search.
PgFSize  If Compute PgeneF: Size of the PgeneF which contains this hit. Sort on this column to view the other rows with same PgeneF.
Hit#  The hit number, which are sequential numbers for the chromosome pair. This is shown on the "Chromosome Explorer" when the mouse is over the hit line.
Hit %Id  Percent identity of the alignment.The value of the "Identity" column from the MUMmer file.
Hit %Sim  Percent similarity of the alignment (as determined by the BLOSUM scoring matrix). The value of the "Similarity" column from the MUMmer file.
Hit #Merge  The number of clustered hits into one merged hit.
Hit St  If "=", both hit ends are to the same strand; if "!=", they are to different strands.
Location: one row for each species
Chr  Chromosome (or draft sequence number) where the hit is aligned
Gstart/Gend/Gs  Gstart/Gend of the annotated gene. The Gst is the strand (+/-).
Gene#  The gene number is C.#.{a-z}. The C is the chromosome number. The # is the sequential number along the chromosome. If a run of genes overlap, they receive the same gene number with different suffixes {a-z, a2-z2, etc}. This is shown on the Chromosome Explorer "Annotation Description" and when the mouse hoovers over the gene.
Hstart/Hend  Hstart/Hend of the hit region.
Merged hits: Hits are clustered during anchor loading, so hit regions are not necessarily single MUMmer alignments.
Annotation: one row for each species
The keywords for the annotations of each species are listed. They can be different for each species, e.g. one may have "ID" while another uses "Name" for the gene names. "All_anno" shows all of the attributes joined together. The keywords used can be limited using the Project Parameters (see annot_kw_mincont and annot_keywords).

Statistics

This section shows overall statistics for the query results. Most of the statistics are self-explanatory except the following:

Annotated and Genes: The first is the number of hits that overlap one or more genes, where a gene can have multiple hits. The second is the number of Genes with at least one hit.

Regions: This statistic is only shown if Compute PgeneF was checked, and it is the number of distinct regions covered on that species.
