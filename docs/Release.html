Release: v5.7.8 (3-Dec-25)

Release: v5.7.2 (2-Sept-25)

The synteny algorithm has new Strict that greatly improves synteny blocks for two complete genome sequence synteny.

  • Synteny computation: See Pair parameters.
    • Strict New: This option provides the following changes to the original algorithm:
      • The gap size is smaller and the PCC (pearson correlation coefficient) is larger.
      • It first creates blocks with the same orientation, but then will merge contained blocks.
    • Merge: This now has three options: None, Overlap, Close.
    • Original: If Strict is not selected, the algorithm is referred to as Original and gives the exact same results as before. However, the Merge requires the blocks to be closer than before in order to be merged.
  • A&S:
    • Self-synteny: Bug fixes (1) v570 Bug fix: the chrN-chrN was broken. (2) Forever Bug: When running self-synteny, it could on rare occasion exit with error; this has been fixed.
    • If Verbose, the number of collinear sets not in blocks is part of the terminal/log output now.
    • Stop writing files anchors.txt and blocks.txt since this info is available from Queries and Report.
    • Clear pair: The button become deselected when clearing and there is output to the terminal when done.
  • Report: See Report.
    • Blocks plus now shows the AvgGap1, AvgGap2, StdDev1, StdDev2. Release v5.7.1 had an approximation for AvgGap1, AvgGap2, where now it is exact. It is necessary to run the synteny algorithm again to get these changes.
    • The Corr (correlation coefficient) column was sometimes a little off if a block merge had occurred.
    • For self-synteny, the blocks will be from one side of the diagonal so as not to be repeated.
  • Small display changes:
    • Queries: The table will show the hit St signs column as "==" or "!=".
    • 2D: The Sequence filter has a new option Block# 1st, which will only place the Block# near the first hit of the block.
    • Dot plot (Minor): The 2D problem of not displaying - I thought I fixed in v571 but did not. Now it is 'sort of' fixed as the panel displays and then disappears if the block was not correctly selected.
    • Summary: Could get a negative number for a percentage.

    Release: v5.6.0 (23-Jan-25)

    There were some major changes made to Algo2 (first released v5.4.6) and other minor changes.
    For existing databases, see Update in the items below.
    • A&S Algo2:
      • The improved algorithm reduces:
        1. the number of long clustered hits with huge gaps (happens with mammalian genomes),
        2. short hits,
        3. hits with little overlap to connected genes (this last change causes much smaller Summary GNwCH and GNwCHB numbers).
      • G2 assignments do much better at assigning the hit's genes.
        G2 no longer allows wrong strand hits, but the "-wsp" remains to print the wrong strand hits.
        G1 and G0 assignments now require less gaps between hits and a 'Hit len' ratio of > 0.5.
        G1 no longer has minor assignments.
      • Internal parameters are mostly set by automatic analysis of the gene models. The user parameters have been changed from Minimal matched bases to Scale, which is more intuitive to change, i.e. increase filters more, decrease filters less. See Algo2 Hints for tested datasets and parameter changes.
      • The gene overlap score (shown on the Gene popup) has been changed to be the overlap between the hit and the gene without subtracting gaps (same as Algo1).
      • The changes do not change the synteny blocks very much, however, the reduction in short F+ hits can result in slightly better synteny blocks and less tiny blocks.
      Update: The A&S needs to be re-run to get this update.

    Release: v5.4.0 (6-Apr-23)

    This release focuses on improving the detection of gene homology in the Synteny computation.
    For existing databases, rerun Alignment&Synteny using existing MUMmer files.
    • Synteny:
      • Creating clustered hits has been improved for long genes.
      • Filtering hits has been updated to give preference when at least one end is a gene.
      • Details of changes:
        → Long hits with embedded genes are split, which helps detect hits to the embedded genes (15Oct24 disabled since graphics was fixed to show these).
        → When a candidate gene is created from hits, it is checked that it does not span a gene with no hits. Candidate genes used to be created using the maximum length of the annotated genes; this is now maximum size 100k.
        → For finding the top N hits, the Max of query and target(summed subhits (minus overlaps)) of a clustered hit are used instead of summed query subhits.
        → A clustered hit could have a mix of subhits that were on opposite strands and the same strand; this does not happen anymore. (This is rare.)
        → %Identity was just using the query match; now it is Max(pct(query), pct(target)) where pct is the approximate average %identity of the subhits.
        → %Similarity was the value of the first subhit of a clustered hit; now it is computed like the %Identity.
        → Pair Parameters: The TopN and MinDots parameters are checked for integer and >0. When A&S is executed, the non-default parameters are printed to the terminal immediately.
      • These changes increased the number of gene homology hits for Humans to Mouse (Chrs 5,17,X) from (68.6%, 57.0%) to (79.9%,66.3%), and for Humans to Chimpanzee (Chrs 5, X) from (53.9%, 54.6%) to (69.9%, 56.7%).
      • Note, these changes cause more hits to be saved, and hence, the synteny takes a little longer. It may also remove or add small blocks due to additional intervening hits.
    • Summary: See Summary.
      • The summary has been rewritten in a textual format which can be copied with copy & paste.
      • All hit and block statistics were computed in terms of clustered hits, which may have gaps and overlaps between the subhits. Now many of the statistics are in terms of the subhits, and it is clear as to whether clustered hits or their subhits are being used.
